2. **Prefetch transcripts (paced, block-safe):** `prefetch-transcripts.py <N> <queuefile>` → caches to `scripts\transcripts\<vid>.txt`. **A fresh Proton IP yields only ~16–22 fetches before YouTube rate-blocks it** ("blocking requests from your IP"); the block is often **short-lived (~70s)** so a re-run on the SAME IP recovers the failed tail. When an IP stays blocked, **rotate Proton to a fresh/less-common server** — popular hubs (US/Canada) get re-blocked fast; **small countries (e.g. Belarus) and P2P servers are far less YouTube-blocked**. `yt-dlp` is NOT a usable fallback under a block (it gets HTTP 429 / bot-check on the same IPs). **Put low-priority vids EARLY in the queue, not last** (last-in-queue vids always eat the rate-limit). oEmbed metadata (`get-yt-meta.py <queue> --cached-only`) still works under a transcript-API block.
